# Prototypes

This folder is for standalone `.html` screen-design prototypes.

These files are allowed to be exploratory. Their job is to make layout, component shape,
and data needs visible before the final frontend/backend structure is built.

## Suggested Prototype Order

1. `my-status.html`
2. `projects.html`
3. `project-detail.html`
4. `quest-board.html`
5. `life-todo.html`

## Prototype Review Checklist

- Does the screen match the LifeGame visual direction: bright, simple, modern, not overly decorative?
- Which components are repeated?
- Which fields does each component need?
- Which ERD tables provide those fields?
- Which fields are missing from ERD v0.2?
- Which API endpoint would naturally feed the screen?
